## Постановка задачи

Детектирование — задача в которой требуется определить:

1.) Ограничивающий прямоугольник (bounding box) в который заключен объект;

2.) Класс объекта.

Прямоугольник (bounding box) задается 4 числами (координатами двух углов, либо координаты центра + высота и ширина). Обычно все эти занчения нормируются на размер изображения.

Раз нужно предсказывать вещественные числа, то к задаче классификации добавляется регрессионная. И лосс функция будет вклюяать два компонента:

Loss = L_class + aL_reg

Так как количество объектов на изображении заранее неизвестно, то детекторы возвращают предсказания для избыточного количества bounding box большая часть их которых впоследствии отбрасывается.

Во многом различия детекторов обусловленны тем как делаются начальные приближения для координат этих bounding box.

Общая схема работы детектора:

    Изображение единожды пропускается через базовую часть сети (encoder), дальнейшие предсказания делаются на карте признаков.

    Во всех случаях предсказываются смещение и масштабирование для финальных bounding boxes.

    Для каждого bounding box предсказывается класс.

    Предсказывается заведомо больше bounding boxes, чем может быть объектов на изображении.


## История детекторов

**Чтобы** лучше понять работу современных детекторов, полезно знать, как они эволюционировали.

Сначала для поиска регионов, в которых может быть расположен объект (ROI — Region Of Interest), использовалась эвристика. Для каждого ROI запускалась нейросеть, предсказывающая: а) класс объекта; б) смещение ROI (xyxy) относительно исходного положения, выданного эвристикой (RCNN).

Чтобы ускорить процесс, ROI стали проецировать на карту признаков и делать предсказание уже на признаках (features) (Fast-RCNN). Для проекции использовались слои **ROI Pooling**, а затем **ROI Align**.

Эвристику заменили на отдельную нейросеть, которая предсказывала ROI сразу на признаках (Faster-RCNN).

Позже нейросеть для предсказания ROI убрали, заменив её фиксированным набором прямоугольников (~10K) (One Shot Detector).

## Loss для детектора

Лосс функция для детекторов включает две компоненты: регрессионную (MSE) оценивающую качество предсказания координат ограничивающих прямоугольников (bounding box) и классификационную (CE) - отвечающую за предсказания класса объекта внутри bounding box. Суммарный лосс это их взвешенная сумма: **Loss = L_class + aL_reg**

Для классических детекторов проблемой тако лосс функции является наличие огромного количества bounding box которы не попадают на объект. Поэтому bbox у которых пресечение с GT(истинный bbox из разметки) исключаются из подсчета лосс.

Другой проблеммой является сильный дисбалланс классов: как правило большая часть пикселей изображения относиться к классу background. И даже если ошибка модели для этих пикселей невелика, то сумма этих небольших ошибок становиться значительной.

Что бы нивелировать их влияние используют FocalLoss

## mAP - метрика для оценки детекторов

Чтобы подсчитать классификационные метрики (Precision, Recall, F1-Score), основанные на подсчёте ошибок I и II рода (TP, FP, FN, TN), необходимо выбрать способ сопоставления предсказанных моделью bbox с bbox из GT (Ground Truth).

При расчёте mAP задаётся порог IoU, и все bbox'ы с уровнем пересечения ниже этого порога не считаются ошибками. На основе такой разметки стандартным образом строится PR-кривая (Precision-Recall Curve).

Затем сглаживаем её (красный пунктир). Площадь под этой кривой и будет значением метрики AP (Average Precision). Если повторить эту процедуру для всех классов и усреднить, получим значение mAP (mean Average Precision) — метрику, которая оценивает качество детектирования для выбранного порога по IoU.

сравнивается с эмбеддингами bbox

## DINO + SAM

Эта связка позволяет автоматически находить и выделять объекты на изображении **без ручной разметки** и **без предварительного обучения на конкретной задаче** — всё работает "из коробки", благодаря возможностям современных foundation-моделей.

Как они работают вместе?

[**Grounding DINO**](https://github.com/IDEA-Research/GroundingDINO) получает на вход изображение и текстовый запрос (например, *cat*, *person*, *car*) и возвращает **bounding boxes** — области изображения, которые с высокой вероятностью соответствуют запрошенному объекту.

[**SAM**](https://github.com/facebookresearch/segment-anything) использует эти bounding boxes как подсказки и на их основе строит **маски сегментации**, отделяя найденные объекты от фона на уровне пикселей.

Материалы: [DINO_SAM.ipynb](DINO_SAM.ipynb), [SAM.ipynb](SAM.ipynb)
