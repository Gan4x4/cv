## Нейронные сети как модели обучения представлений

Качество работы моделей классического машинного обучения напрямую зависит от того, как мы представим данные на входе модели.

Мы можем увидеть эффект от удачных представлений, рассмотрев иллюстрацию ниже. 

![](https://ml.gan4x4.ru/msu/dev-2.2/L12/out/two_circles_representation.png)


В этом классическом примере мы хотим обучить модель машинного обучения, скажем, логистическую регрессию, для поиска прямой линии, разделяющей голубые и оранжевые точки, описываемые признаками 𝑥1 и 𝑥2 .

Модель, которая способна строить лишь линейные разделяющие поверхности, не справится с этой задачей, потому что невозможно разделить два класса с помощью прямой, используя данные, представленные в таком виде.

Если мы изменим представление данных для этой модели и будем подавать ей на вход квадраты центрированных исходных признаков, то мы увидим, что ландшафт данных станет совершенно иным и линейная модель сможет разделить объекты по классам.

## Metric learning

Подход в машинном обучении, направленный на обучение моделей, которые могут измерять "сходство" или "различие" между объектами.

![Metric learning](https://ml.gan4x4.ru/msu/dev-2.2/L12/out/siamese_neural_network_scheme.png)

Два входных изображения (𝑥1 и 𝑥2) проходят через одну и ту же сверточную сеть, на выходе для каждого изображения генерируется вектор признаков фиксированной длины (ℎ1 и ℎ2 соответственно).

Модель обучается генерировать близкие вектора для изображений одной персоны и далекие для разных.

Оценивая расстояние между двумя векторами признаков, которое будет малым для схожих объектов и большим для различных, мы сможем оценить их сходство. Это центральная идея сиамских сетей.

Материалы: [Metric_learning.ipynb](Metric_learning.ipynb), [presentation](https://docs.google.com/presentation/d/1L2YioPBnV1kxQ4YkPLtqHP_DxxRJZALQrie2uHJpofs/edit?slide=id.p#slide=id.p)

## Автоэнкодеры

Архитектура нейросети, сжимает признаковое описание объекта в вектор небольшой размерности, а затем восстанавливает этот вектор в исходное признаковое пространство.


![Autoencoder](https://ml.gan4x4.ru/msu/dev-2.2/L12/out/autoencoder_architecture.png)


Encoder и Decoder две нейросети. Особенность в том что размерность выхода Encoder(latent) сильно меньше размерности входных данных. Соответственно Decoder содержит разжимающие блоки.

Обучение:

Модель должена быть предсказать входное изображение. Поэтому разметка данных не нужна. В качестве loss используется MSE, или BCE если данные и отображенны в итервал [0..1].

Задачи

С помощью автоэнкодеров становится возможным обучать представления для данных, не имея разметку. Эти представления могут затем использоваться для решения других задач:

- понижение размерности,
- очистка данных от шумов,
- предобучение для дальнейшего дообучения на размеченных данных,
- обнаружение аномалий,

Для генерации новых данных AE подходит плохо, эта задача решается при помощи VAE.

P.S. Мы можем использовать любую сеть для энкодера и декодера в зависимости от природы данных: на полносвязных, свёрточных и даже рекуррентных слоях.

## CLIP (Contrastive Language–Image Pre-training)

Архитектура, разработанная компанией OpenAI, которая обучается соотносить изображения и текст.

![](https://ml.gan4x4.ru/msu/dev-2.1/L08/clip_idea.png)
CLIP преобразует изображения и текст в единое векторное пространство, что позволяет сравнивать их между собой эмбеддинги текста и картинок. Это делает возможным поиск по изображениям с использованием текстовых запросов и наоборот.
